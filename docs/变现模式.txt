2. 产品技术架构
你的产品不再只是一个插件，而是一个Docker 镜像发行版。
* 底层 (OS): 预装 Python/Node/Docker 环境，配置好所有依赖。
* 核心 (Core): opencode.ai CLI + 你的两大插件 (OpenAI/Google 验证)。
* 交互 (UI): 集成 code-server (网页版 VS Code) 或 ttyd (网页终端)，实现浏览器内操作。
* 网络 (Network - 关键):
   * 内置代理客户端（支持 HTTP/Socks5 验证）。
   * 配置好环境变量 RESIDENTIAL_PROXY_URL，让 Docker 的出口流量走“纯净 IP”。
________________
3. 用户分层与转化策略 (The Funnel)
不要把所有用户一视同仁，要根据他们的需求进行分流：
A 类用户：API Key 土豪党 (最省事)
* 特征： 直接用 OpenAI/Claude 的官方 API Key，不在乎钱，只求稳。
* 痛点： 本地电脑不能关机，任务跑一半断网。
* 你的方案： “云端永生模式”。
* 转化路径： 推荐部署到 DigitalOcean ($200 免费额度)。
* 话术： “一键部署，24/7 在线，任务永不中断。”
B 类用户：会员权益白嫖党 (你的主力军)
* 特征： 买了 ChatGPT Plus/Gemini Advanced，想用你的插件白嫖 API 能力。
* 痛点： 云端部署会被封 IP，本地部署电脑风扇狂转且不能关机。
* 你的方案： “云端伪装模式”。
* 转化路径： 推荐部署到 DigitalOcean + 购买静态住宅代理。
* 话术： “搭配静态住宅 IP ($3/月)，让云端助手拥有家庭宽带身份，零风险解锁会员权益，比官方 API 省钱 90%。”
C 类用户：白嫖极客党 (你的口碑来源)
* 特征： 既不想买服务器，也不想买 IP，动手能力强。
* 你的方案： “纯净源码版”。
* 策略： 提供源码，但不提供一键配置。文档里写明复杂的依赖安装步骤。
* 作用： 他们的存在证明了你的项目是“真开源”，防止社区 Fork。同时他们会在论坛帮你吹牛。
